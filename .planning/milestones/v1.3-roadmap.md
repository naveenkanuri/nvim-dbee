## Scope

v1.3 restores trust in shipped v1.1/v1.2 workflows first, then addresses the highest-impact enterprise database UX gap: huge schema sets that need explicit schema scoping and progressive loading.

Keep:
- v1.0/v1.1/v1.2 locked decisions D-01..D-197.
- Phase 7 connection-only drawer and handler-owned lifecycle contracts.
- Phase 8 type-aware wizard and FileSource atomic-write contracts.
- Phase 10 LSP perf harness and Phase 11 async completion/cache/diagnostics contracts.
- Singleton-per-current-connection built-in dbee LSP model.

Do not:
- Reopen shipped drawer, wizard, handler, reconnect, or LSP architecture without a Phase 14 discuss lock.
- Duplicate Phase 9 drawer perf infrastructure or Phase 10 LSP perf infrastructure.
- Fix pre-existing `a and nil or b` pattern bugs or the four pre-existing legacy headless failures in v1.3.
- Add semantic tokens, inlay hints, `vim.lsp.config()` migration, or multi-client LSP architecture.

## Phase 14: Enterprise DB UX Architecture

**Goal:** Make enterprise databases with huge schema/table sets usable through per-connection schema allowlists and schemas-only initial structure loading with per-schema lazy table fetch.  
**Depends on:** Phase 13, Phase 11 LSP optimization/correctness infrastructure.  
**Risk:** High. Architecture change across wizard, handler, drawer, cache, and LSP.  
**Requirements:** DBEE-ARCH-01

**Success criteria:**
1. FileSource-backed connection JSON can persist additive `schema_filter` metadata with exact names and prefix globs; absent filter preserves current all-schema behavior.
2. Wizard add/edit can discover schemas after a non-mutating probe and supports manual entry when discovery fails.
3. Initial structure loading can fetch schemas only; expanding a schema fetches that schema's tables/views asynchronously with a row-local loading state.
4. Drawer, LSP completion, LSP diagnostics, schema cache, and disk cache all honor the same schema allowlist.
5. LSP `schema.` completion remains non-blocking: cache hits return immediately, first table-list miss warms asynchronously and reports `isIncomplete` only when truthful.
6. Cache invalidation on `schema_filter` changes clears affected drawer/LSP state without stale completions or diagnostics.

**Research bullets:**
- Revisit Phase 6 D-30..D-63 because Phase 14 intentionally deepens the root/schema/table loading model that Phase 6 caveated.
- Preserve Phase 7 D-77/D-78/D-84/D-87 handler-owned single-flight, authoritative root epoch, backpressure, and waiter cleanup.
- Preserve Phase 11 D-154..D-161 async completion miss discipline; do not reintroduce synchronous metadata fetches for `schema.` table completion.
- Decide in discuss whether unfiltered-schema diagnostics should warn, skip, or surface as out-of-scope/noise.
- Decide exact `schema_filter` metadata shape, glob matching semantics, and case sensitivity.

## Phase 15: LSP + Drawer Polish Batch

**Goal:** Clean up residual LSP/cache and drawer UX issues after Phase 14 settles the schema/cache/loading shape.  
**Depends on:** Phase 14.  
**Risk:** Low-medium. Targeted cleanup, but touches hot LSP indexes and drawer rendering.  
**Requirements:** DBEE-POLISH-01

**Success criteria:**
1. Incremental `schema_lookup` updates are rebuild-equivalent when schemas differ only by case.
2. Targeted global table-index updates avoid walking unrelated schemas by maintaining an incremental representative map.
3. Connection source badges render only when more than one source needs disambiguation.
4. Cold drawer start has a lightweight orientation affordance without reintroducing source parent rows.
5. Long loading states show elapsed time after a threshold, can be cancelled manually, and fail with a clear timeout/error after a configurable ceiling.

**Research bullets:**
- Re-read Phase 11 D-165..D-167 and D-177..D-178 before changing schema lookup/index behavior.
- Re-read Phase 7 D-65 before changing connection-row source badge presentation.
- Re-read Phase 6 D-36 and Phase 14 loading shape before designing timeout/cancel behavior.

## Phase 16: LSP Feature Gap Closure

**Goal:** Add high-value LSP features that can be supported truthfully from schema/cache state.  
**Depends on:** Phase 15.  
**Status:** Conditional. Decide at Phase 15 ship.  
**Risk:** Low-medium. Additive features.  
**Requirements:** DBEE-FEAT-02

**Candidate scope:**
- `completionItem/resolve` with lazy details.
- Hover for table/column metadata.
- Code actions for schema refresh/reload on stale schema state.
- Document/workspace symbols for cached schema objects.

**Deferred explicitly:**
- Semantic tokens.
- Inlay hints.
- `vim.lsp.config()` / `vim.lsp.enable()` migration.
- Multi-client or per-buffer connection architecture.

**Success criteria:**
1. Feature work remains additive and does not reopen Phase 10/11 perf/correctness contracts.
2. Resolve/hover/symbol responses are cache-backed and truthful when data is unavailable.
3. Code actions trigger existing refresh/reload paths rather than inventing parallel invalidation flows.
