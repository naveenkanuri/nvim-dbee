# Phase 12.3: LSP Code Actions - Plan

**Created:** 2026-05-01
**Status:** Ready for plan-gate review r3
**Base HEAD:** `692f64a`
**Depends on:** Phase 12.1 and Phase 12.2 shipped

## Goal

Ship safe cache-backed LSP code actions:

- Cursor on `*` in `select * from public.users` offers `Expand SELECT * -> list columns` and replaces `*` with cached columns.
- Cursor on `users` in `select id from users` offers `Qualify identifier: users -> public.users`.
- Normal code-action requests offer `Refresh schema cache`; table ranges also offer `Reload table metadata`.
- All actions fail closed when authority, epoch freshness, scope, or range safety is unavailable.

## Upstream Decisions

This plan inherits D-306..D-450 from Phase 12 umbrella, 12.1, and 12.2 artifacts.

## Phase 12.3 Decisions

- **D-451:** Phase 12.3 ships only standard LSP code actions plus `workspace/executeCommand` for server-owned commands.
- **D-452:** Advertise `codeActionProvider` only when Phase 12.3 code actions are enabled.
- **D-453:** Advertise `executeCommandProvider.commands` as the exact enabled command-action subset. If only refresh is enabled, advertise only `dbee/refresh_schema`; if only reload is enabled, advertise only `dbee/reload_table`; if both command actions are disabled, omit `executeCommandProvider` while leaving edit code actions available.
- **D-454:** Do not add `codeAction/resolve` in Phase 12.3.
- **D-455:** `textDocument/codeAction` returns stable ordered `CodeAction` literals, not client-local Neovim commands.
- **D-456:** Add `lsp.code_actions = true` as the master rollback flag.
- **D-457:** Add flat per-action toggles: `lsp.code_action_expand_select_star`, `lsp.code_action_qualify_identifier`, `lsp.code_action_refresh_schema`, and `lsp.code_action_reload_table_metadata`, all default `true`.
- **D-458:** Add `lsp.code_action_max_expand_columns = 200`; omit expand action for wider tables.
- **D-459:** Implement a new `lua/dbee/lsp/code_actions.lua` registry.
- **D-460:** Stable picker order is expand star, qualify identifier, reload table metadata, refresh schema cache.
- **D-461:** Honor `context.only` filtering.
- **D-462:** Refresh is range-independent in normal requests; `LSP12_3_NO_ACTIONABLE_RANGE_EMPTY` applies to refactor-only requests outside actionable ranges.
- **D-463:** Expand applies only to an unqualified top-level select-list `*`. Qualified wildcard forms such as `u.*`, `"u".*`, `schema.table.*`, or any `*` immediately preceded by a dot-qualified identifier return no action in v1.3.
- **D-464:** Expand supports single-table statements only in v1.3.
- **D-465:** Expand uses already-cached, helper-routed, fresh, in-scope column metadata.
- **D-466:** Edit actions emit one current-document text edit inside `WorkspaceEdit.documentChanges` with an `OptionalVersionedTextDocumentIdentifier`; the server tracks the synced document version from `didOpen`/`didChange` and omits edit actions when no current version is known.
- **D-467:** Generated columns preserve quote intent and use `schema_name_canonical` to decide safe unquoted exact names.
- **D-468:** Qualify applies only to an unqualified table identifier from table-reference context.
- **D-469:** Already-qualified identifiers, aliases, columns, keywords, CTE/local relation shadows, and ambiguous global matches return no action.
- **D-470:** Qualify prepends the resolved schema only; it preserves the existing table spelling.
- **D-471:** Schema rendering uses the same quote-preserving renderer, never local fold logic.
- **D-472:** Refresh command id is `dbee/refresh_schema` with active connection and freshness tokens.
- **D-473:** Reload command id is `dbee/reload_table` with active connection, schema/table, quote flags, and freshness tokens.
- **D-474:** `workspace/executeCommand` accepts only the enabled command ids in the advertised provider subset.
- **D-475:** Command execution rechecks the per-command enabled flag, active connection, scope, generation, and root epoch.
- **D-476:** Commands return immediately after scheduling async work.
- **D-477:** Schema refresh routes through existing active-connection LSP root-refresh flow.
- **D-478:** Table reload adds a small schema-cache command helper if needed to force async column reload while preserving epoch write admission.
- **D-479:** Authority-unavailable returns no actions; legacy authority API absence keeps implicit-all behavior.
- **D-480:** Stale root epoch returns no actions.
- **D-481:** Unsupported, unknown, filtered, missing, or ambiguous contexts return empty lists.
- **D-482:** Text edits are single-document, single-range, versioned `documentChanges` only. A stale client-side apply after the buffer changes must reject or no-op instead of applying to a shifted range.
- **D-483:** Multi-statement same-line SQL uses Phase 12.2 statement splitting.
- **D-484:** New code-action code must not bypass schema-filter, canonical-name, or epoch helpers, and must not call request-path `connection_get_*`.
- **D-485:** Revision 0 marker ledger was 31 rollup-checked markers and 33 emitted markers. Revision 1 supersedes this with D-499.

## Revision 1 Plan-Gate Closure Decisions

- **D-486:** Edit actions require versioned document edits. The response shape is `WorkspaceEdit.documentChanges = { { textDocument = { uri = <uri>, version = <number> }, edits = { <single edit> } } }`, not `WorkspaceEdit.changes`. Neovim's workspace-edit application is the apply-time version gate; the server's responsibility is to maintain synced document versions and include the current version in each edit action.
- **D-487:** `LSP12_3_STALE_EDIT_REJECTED` proves a code action captured at document version N does not modify the buffer after the buffer advances to N+1.
- **D-488:** Qualified wildcard expansion is deferred. Phase 12.3 rejects `alias.*`, quoted-alias `*`, and multi-part qualified `*` ranges rather than expanding the `*` token alone.
- **D-489:** Add token-aware CTE/local relation extraction for `WITH name AS (...)` scopes. If the selected table identifier matches a CTE/local relation name, qualify and reload actions return no action. If a statement contains CTE syntax but extraction cannot prove that the selected relation is safe, fail closed.
- **D-490:** Direct execute-command calls are governed by the same per-action toggles as code-action discovery. A disabled command action is not advertised, is not returned by `textDocument/codeAction`, and is rejected when invoked directly.
- **D-491:** Refresh/reload command ownership is explicit: `lua/dbee/lsp/init.lua` owns the active-connection callbacks; `server.lua` only routes LSP protocol; `code_actions.lua` only builds actions, validates command ids, and invokes injected callbacks.
- **D-492:** The helper-bypass static guard scans new `init.lua` command callback bodies in addition to `code_actions.lua`, new context helpers, schema-cache reload helpers, and server execute-command dispatch.
- **D-493:** `LSP12_3_RELOAD_CMD_IMMEDIATE_ASYNC` proves reload schedules async column refresh and returns before any column result callback/write admission completes.
- **D-494:** `context.only` uses LSP hierarchical kind-prefix semantics: `refactor` includes `refactor.rewrite`, `source` includes source actions, exact `refactor.rewrite` includes only rewrite refactors, and mixed arrays preserve stable ordering across all included actions.
- **D-495:** Perf gates are computed from actual `textDocument/codeAction` request timings for all four Phase 12.3 cohorts, with per-cohort P95 markers emitted. Helper-only timings are not sufficient evidence.
- **D-496:** The expand-star perf fixture uses a table at `lsp.code_action_max_expand_columns` to measure the worst allowed edit construction path; the wide-table sentinel instruments copy/render code and fails if an over-cap table is copied or rendered before omission.
- **D-497:** Wave sequencing prevents half-integrated advertisement: Wave 1 may add handler scaffolding and config parsing, but default-enabled `codeActionProvider` / `executeCommandProvider` advertisement is completed only after enabled action registry and command subset logic exist.
- **D-498:** Plan-gate r1 blockers and watchpoints are all plan-owned and sentinel-covered or explicitly folded into task/wave/perf text. No r1 item is deferred to v1.4 except qualified wildcard expansion, which is intentionally rejected in v1.3.
- **D-499:** Revision 1 marker ledger was 38 rollup-checked markers and 40 emitted Phase 12.3 markers. Revision 2 supersedes this with D-503.

## Static Guard Plan

`LSP12_3_NEW_CODE_ACTION_NO_HELPER_BYPASS` must scan:

- `lua/dbee/lsp/code_actions.lua`
- new code-action helper bodies in `lua/dbee/lsp/context.lua`
- new reload helper body in `lua/dbee/lsp/schema_cache.lua`
- server execute-command dispatch in `lua/dbee/lsp/server.lua`
- new refresh/reload command callback bodies in `lua/dbee/lsp/init.lua`

Reject:

- direct underlying schema-filter API calls (`get_schema_filter_normalized(`) or local schema-filter interpretation from new code-action paths,
- direct `epoch_authority.handler_epoch`, `cache_epoch`, or `read_with_freshness` in `code_actions.lua` except through cache-provided freshness token helpers,
- local lower/upper canonical matching in new code-action lookup/render paths,
- request-path `connection_get_*`,
- direct handler metadata calls from new `init.lua` command callback bodies except the approved `_request_root_refresh` refresh path,
- command ids outside `dbee/refresh_schema` and `dbee/reload_table`.

Allow:

- helper API calls such as `schema_filter_authority.read` / `schema_filter_authority.is_fail_closed`,
- existing schema-cache helper internals that already call authority/epoch helpers,
- command-path async scheduling from the named reload helper and init refresh callback.
- command callback invocation through injected `code_action_commands` only; `code_actions.lua` must not require or mutate `dbee.lsp.init`.
