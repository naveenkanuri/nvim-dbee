# Phase 12.2: LSP Document And Workspace Symbols - Plan

**Created:** 2026-05-01
**Status:** Ready for impl-gate r5
**Depends on:** Phase 12.1 at `4954dca`

## Goal

Ship cache/source-truthful LSP symbols:

- User opens `select * from users join orders on users.id = orders.user_id`; `textDocument/documentSymbol` returns `users` and `orders` with valid source ranges.
- User runs workspace symbol search; `workspace/symbol` returns known active-connection schemas/tables from the cache, filtered by schema authority and protected by epoch freshness.

## Upstream Decisions

This plan inherits D-306..D-337 from `.planning/DISCUSS-phase12.md` and D-338..D-395 from `.planning/phases/12-lsp-feature-gap-closure/12-01-PLAN.md`.

## Phase 12.2 Decisions

- **D-396:** Phase 12.2 ships only `textDocument/documentSymbol` and `workspace/symbol`. Code actions remain Phase 12.3.
- **D-397:** Phase 12.2 uses standard LSP protocol methods only. No custom Neovim picker or cmp-only API is introduced.
- **D-398:** Add rollback flags `lsp.document_symbols=true` and `lsp.workspace_symbols=true`.
- **D-399:** Advertise `documentSymbolProvider=true` and `workspaceSymbolProvider=true` only when corresponding flags are enabled.
- **D-400:** `textDocument/documentSymbol` returns hierarchical `DocumentSymbol[]` when `hierarchicalDocumentSymbolSupport` is true, otherwise flat `SymbolInformation[]`.
- **D-401:** Document symbol hierarchy is `Schema -> Table -> Column` when a schema qualifier exists. Unqualified or unknown-schema table references are emitted as root-level table symbols with no synthetic parent. Columns are emitted only when unambiguously tied to a table.
- **D-402:** Document symbol ranges are source identifier ranges. Repeated references are deduped by canonical identity and keep the first occurrence range.
- **D-403:** Document symbols parse the whole document up to 10,000 lines or 1 MiB. The hover 200-line cursor scan is not reused as the primary document-symbol scan.
- **D-404:** Document symbols reflect SQL source references even when cache metadata is missing, filtered, stale, or authority-unavailable. Optional cache enrichment must degrade to name-only; authority never suppresses source-text document symbols.
- **D-405:** Document-symbol parse results may be cached per URI/bufnr/changedtick/response-mode and must be invalidated by `didChange`, `didClose`, buffer deletion, or changedtick mismatch.
- **D-406:** Workspace symbols search only the active/current connection.
- **D-407:** Workspace symbols must route cache reads through `schema_filter_authority` and `epoch_authority`. `authority_unavailable` returns an empty list, `api_absent_legacy` preserves implicit-all behavior, and `ok` applies the returned scope.
- **D-408:** Workspace symbols are cache-only and must not call synchronous or asynchronous DB metadata APIs.
- **D-409:** Workspace query matching uses `schema_name_canonical` for case-insensitive canonical substring matching. Ordered-character relaxed matching is deferred to v1.4 and is not a Phase 12.2 requirement.
- **D-410:** Workspace symbol results are deterministically capped at 200 by default. Filtering and cap selection happen on a compact index before allocating LSP symbol result records. LSP has no `workspace/symbol` `isIncomplete` result wrapper, so Phase 12.2 does not emit one.
- **D-411:** Workspace symbols are flat. Prefer `WorkspaceSymbol[]`; fall back to `SymbolInformation[]` with `containerName`, percent-encoded synthetic `dbee://` URI, and zero range.
- **D-412:** Symbol kind mapping is Schema=`Namespace(3)`, Table=`Class(5)`, Column=`Field(8)`.
- **D-413:** Returned symbol display names preserve exact cache/source names. Matching and dedupe never use local fold logic.
- **D-414:** No Go adapter capability or metadata contract changes are introduced.
- **D-415:** Performance budgets are hard gates: documentSymbol P95 <50ms and workspace/symbol P95 <100ms on a 10,000-table cache. Budget markers must be `true`, not `unfrozen`.
- **D-416:** Phase 12.2 perf probes emit `LSP12_2_PERF_SCENARIOS_COUNT=5` and do not change Phase 12.1 `LSP12_PERF_SCENARIOS_COUNT=10`.
- **D-417:** Phase 12.2 rollup requires every strict marker exactly once, following D-377/D-395.
- **D-418:** Symbols must not introduce direct epoch reads outside `epoch_authority`, direct schema authority interpretation outside `schema_filter_authority`, or local canonical folding outside `schema_name_canonical`.
- **D-419:** Symbols are name-only. No markdown rendering or object documentation is added in Phase 12.2.
- **D-420:** If a document exceeds parse caps, return only the truthful symbols found within the cap and keep the handler bounded.
- **D-421:** Workspace symbol invalidation lag must fail closed the same way hover/resolve/diagnostics do.
- **D-422:** User docs are limited to the two feature flags and high-level LSP symbol behavior.
- **D-423:** Verification must run `make perf-lsp PERF_PLATFORM=macos` and `go -C dbee test ./core ./handler ./adapters`.

## Revision 1 Gate Closure Decisions

- **D-424:** Document-symbol parsing must use single-pass token/range capture or an equivalent line-offset index. It must not repeatedly rescan prefixes for every symbol range in dense documents.
- **D-425:** The large-buffer document-symbol sentinel uses a near-cap corpus with at least 10,000 lines or near 1 MiB and symbols distributed before and after line 200.
- **D-426:** Unqualified table references such as `users` and unknown qualified references such as `unknown.tablex` are source-truthful root-level table symbols when no trustworthy schema parent exists.
- **D-427:** Document symbols are authority-neutral. Schema-filter authority, stale cache epochs, and cache misses cannot suppress source-text symbols; they only prevent optional cache enrichment.
- **D-428:** Rollback flags are rollup-covered. Disabled document/workspace symbol flags remove initialize capabilities and make direct requests return protocol-safe nil/empty responses.
- **D-429:** `make perf-lsp` must run `ci/headless/check_lsp12_2_symbols.lua` before the LSP12 rollup reads marker output. A plan that adds the script without Makefile wiring is incomplete.
- **D-430:** No-sync/no-async evidence uses request-path counters for `textDocument/documentSymbol` and `workspace/symbol`, plus scoped static guards on `symbols.lua` and newly added symbol snapshot helper bodies only.
- **D-431:** Workspace authority status handling is sentinel-covered for all three statuses: `authority_unavailable`, `api_absent_legacy`, and `ok`.
- **D-432:** Workspace symbol snapshots use a compact index record shape such as `{ schema_canonical, table_canonical, schema_exact, table_exact, kind }`; the query path filters and caps compact records before allocating `WorkspaceSymbol` or `SymbolInformation` tables.
- **D-433:** Synthetic workspace symbol URIs use `dbee://<connection_id>/<schema>/<object>` with percent-encoding for spaces, `/`, `?`, `#`, `&`, `%`, quotes, and other URI-reserved bytes.
- **D-434:** Workspace symbol location fallback is explicit: if no SQL buffer location is available, return an acceptable DB-object location shape using the encoded `dbee://` URI and a zero range for `SymbolInformation` compatibility.
- **D-435:** Phase 12.2 perf markers are hard `true` gates. The rollup must not accept `unfrozen` for `LSP12_2_DOCSYMBOL_PERF_BUDGET_50MS` or `LSP12_2_WORKSPACESYMBOL_PERF_BUDGET_100MS`.
- **D-436:** Phase 12.2 perf cohorts are five separate scenarios: documentSymbol cold parse, documentSymbol cached parse, documentSymbol dense references, workspace/symbol empty-query all-match, and workspace/symbol selective substring query. Each cohort targets 100 measured iterations unless the existing harness has a stricter global sample policy.
- **D-437:** Ordered-character relaxed workspace symbol matching is explicitly deferred to v1.4. Phase 12.2 requires substring matching only.
- **D-438:** Workspace symbol response-shape behavior is sentinel-covered for both preferred `WorkspaceSymbol` output and `SymbolInformation` fallback output.

## Revision 2 Gate Closure Decisions

- **D-439:** `12-02-PLAN.md` must carry SDK-discoverable execute frontmatter matching the Phase 12.1 plan pattern. Before execution, `gsd-sdk query init.execute-phase "12"` must report `12-01-PLAN.md` and `12-02-PLAN.md` in `plans`, with `12-02-PLAN.md` listed as incomplete while Phase 12.2 is unexecuted.

- **D-441:** Document-symbol parse cache eviction must be proven for all invalidation sources: `didChange`/changedtick, `didClose`, and invalid/deleted buffers. A stale cache entry after close or buffer deletion is a correctness failure.
- **D-442:** Unknown qualified references such as `unknown.tablex` are emitted as root-level table symbols, not as children of a synthetic or untrusted `unknown` schema container.
- **D-443:** Phase 12.2 static helper-bypass checks scan both `lua/dbee/lsp/symbols.lua` and the newly added workspace-symbol snapshot helper body in `lua/dbee/lsp/schema_cache.lua` for direct epoch APIs, direct schema-authority interpretation, and local canonical/fold logic. New symbol code must route through `epoch_authority`, `schema_filter_authority`, and `schema_name_canonical`; the sentinel must fail on injected bypasses in either location.
- **D-444:** `12-02-DISCUSS.md`, `12-02-RESEARCH.md`, and this plan must stay aligned after revision. The current Phase 12.2 source of truth is substring-only workspace matching, five perf cohorts, and `LSP12_2_PERF_SCENARIOS_COUNT=5`.

## Revision 3 Impl-Gate Closure Decisions

- **D-446:** Phase 12.2 r3 adds one strict document-symbol sentinel. The locked marker count is `LSP12_2_ROLLUP_MARKERS_CHECKED=50`, with 52 emitted Phase 12.2 markers including `LSP12_2_ROLLUP_MARKERS_CHECKED` and `LSP12_2_ALL_PASS`.

## Revision 4 Impl-Gate Closure Decisions

- **D-448:** Document-symbol cache enrichment is admitted only when the live schema-filter authority status is `ok`, the cache epoch is fresh, and the referenced schema is inside the live authority scope. Authority-unavailable, filtered, stale, or missing-cache states still emit source-text symbols but degrade to root-level name-only table symbols.
- **D-449:** Diagnostics and document symbols share token-aware table-reference extraction. Diagnostics must ignore `from`/`join` text inside quoted strings, Postgres dollar-quoted strings, line comments, and block comments.
- **D-450:** Phase 12.2 r4 adds two strict sentinels and reconciles the marker ledger. The locked marker count is `LSP12_2_ROLLUP_MARKERS_CHECKED=52`, with 54 emitted Phase 12.2 markers including `LSP12_2_ROLLUP_MARKERS_CHECKED` and `LSP12_2_ALL_PASS`.

## Threat Model

| Threat | Risk | Mitigation | Sentinel |
| --- | --- | --- | --- |
| Workspace symbol authority bypass | Filtered schemas leak through search | Route through schema-filter authority; fail closed | `LSP12_2_WORKSPACESYMBOL_AUTHORITY_FAIL_CLOSED` |
| Workspace legacy authority regression | Legacy handler without authority API unexpectedly returns empty symbols | Preserve `api_absent_legacy` implicit-all behavior | `LSP12_2_WORKSPACESYMBOL_AUTHORITY_LEGACY_IMPLICIT_ALL` |
| Workspace scoped authority regression | `ok` authority result ignores allowed-schema scope | Sentinel uses allowed `public` and disallowed private/audit schemas | `LSP12_2_WORKSPACESYMBOL_AUTHORITY_OK_SCOPED` |
| Epoch stale cache leak | Invalidated cache returns stale symbols | Route reads through epoch authority | `LSP12_2_WORKSPACESYMBOL_EPOCH_FAIL_CLOSED` |
| Local case folding | Quoted/case-sensitive objects collapse | Use `schema_name_canonical`; no local lower/upper matching | `LSP12_2_WORKSPACESYMBOL_CANONICAL_LOOKUP` |
| Huge document parse stalls editor | Whole-document symbols exceed budget | 10,000-line / 1 MiB cap, single-pass parser or offset index, parse cache | `LSP12_2_DOCSYMBOL_FULL_DOC_BOUNDED`, `LSP12_2_DOCSYMBOL_LARGE_BUFFER_BOUNDED`, perf marker |
| Source references suppressed by cache/authority state | Document symbol picker hides real SQL references | Document symbols are authority-neutral and name-only when cache enrichment is unavailable | `LSP12_2_DOCSYMBOL_AUTHORITY_NEUTRAL` |
| Authority leaks document-symbol hierarchy | Filtered or authority-unavailable schemas appear as trusted `Schema -> Table` parents | Require live authority `ok`, fresh epoch, and in-scope schema before enrichment | `LSP12_2_DOCSYMBOL_AUTHORITY_DEGRADES_TO_NAME` |
| Unqualified refs cannot fit hierarchy | `users`/`orders` disappear or get fake source ranges | Root-level table fallback with real table range | `LSP12_2_DOCSYMBOL_UNQUALIFIED_FLAT` |
| Unknown qualified refs get fake hierarchy | `unknown.tablex` appears under an untrusted synthetic schema | Root-level table fallback for unknown qualified refs | `LSP12_2_DOCSYMBOL_UNKNOWN_QUALIFIED_FLAT` |
| Closed/deleted buffer cache leak | Picker returns stale symbols for an invalid document | Evict parse cache on `didClose` and invalid buffer access | `LSP12_2_DOCSYMBOL_DIDCLOSE_EVICTS`, `LSP12_2_DOCSYMBOL_INVALID_BUFFER_EVICTS` |
| Ambiguous column symbols | Picker shows misleading column under wrong table | Emit columns only when unambiguous | `LSP12_2_DOCSYMBOL_COLUMN_SCOPE_SAFE` |
| Line-local document parsing | Common formatted SQL misses multiline table/column refs | Semicolon-scoped statement parsing with multiline token/range capture | multiline document-symbol markers |
| Raw semicolon splitting | Semicolons inside strings/comments create fake statements and fake symbols | Quote/comment-aware statement splitting, including dollar-quoted strings | `LSP12_2_DOCSYMBOL_STMT_SPLIT_QUOTE_COMMENT_AWARE` |
| Diagnostic parser sees non-SQL text | Comments/strings produce false schema-aware diagnostics | Diagnostics use token-aware table-reference extraction shared with document symbols | `LSP12_2_DIAGNOSTICS_IGNORES_COMMENTS_AND_STRINGS` |
| Dense document range lookup | Per-symbol prefix rescans make dense documents superlinear | Per-statement offset index for O(1)/bounded range mapping | `LSP12_2_DOCSYMBOL_DENSE_REFS_BOUNDED` |
| Stale document-symbol enrichment | Same unchanged buffer reuses hierarchy from an old cache/connection | Include cache identity, epoch/generation, fold, and scope signature in the parse-cache key | `LSP12_2_DOCSYMBOL_CACHE_KEY_INCLUDES_CACHE_IDENTITY` |
| Non-column select-list tokens | Literals/functions/aliases appear as columns | Emit only simple identifier-shaped select-list items in the single-table fallback | `LSP12_2_DOCSYMBOL_SELECT_LIST_IDENTIFIERS_ONLY` |
| Non-standard workspace pagination | Client receives invalid response shape | Fixed cap, plain result array, no `isIncomplete` | `LSP12_2_WORKSPACESYMBOL_PAGINATION_OK` |
| Workspace result allocation scales with full cache | 10k table search allocates/copies all rows before cap | Compact index, filter/cap before result allocation | `LSP12_2_WORKSPACESYMBOL_CAP_BEFORE_COPY` |
| Malformed synthetic DB URIs | Picker selection or display breaks on spaces/slashes/query/hash chars | Percent-encode URI path components and test fallback shapes | `LSP12_2_WORKSPACESYMBOL_DBEE_URI_PERCENT_ENCODED`, `LSP12_2_WORKSPACESYMBOL_LOCATION_FALLBACK_OK` |
