# 20-01 - Live PostgreSQL Rich Metadata Smoke

<objective>
Create a focused live PostgreSQL smoke lane that runs Phase 17 rich metadata SQL against a real digest-pinned PostgreSQL 16 container, catches the composite-FK SQL-shape class that sqlmock missed, and exposes deterministic `LIVE_PG20_*` evidence through `make live-pg-smoke`.
</objective>

<must_haves>
- Do not modify `dbee/adapters/postgres_driver_rich_metadata.go`.
- Do not modify locked helpers: `lua/dbee/schema_filter_authority.lua`, `lua/dbee/schema_name_canonical.lua`, `lua/dbee/lsp/epoch_authority.lua`.
- Use testcontainers-go and the existing adapter connection path, not hand-written `podman run` / `docker run` lifecycle code.
- Use digest-pinned PG16 as the primary image: `postgres:16-alpine@sha256:4e6e670bb069649261c9c18031f0aded7bb249a5b6664ddec29c013a89310d50`.
- Keep live-smoke rollup separate from `UX13_ROLLUP_LOG`.
- Sequence assertions cover `Name`, `Schema`, `Increment`, and `CacheSize` only. Do not add or assert `current_value`.
- Live smoke file must use `//go:build live_pg20` and `_smoke_test.go`; bare `go test ./tests/integration` must not compile/run it.
- Negative SQL must use a dedicated `*sql.DB` and exact historical broken FK SQL from `be58045^`.
- `LIVE_PG20_STRICT_MARKER_COUNT=18` is the count contract.
</must_haves>

<strict_markers>
Strict records: `LIVE_PG20_RUNTIME_DETECTED_OK=true`, `LIVE_PG20_CONTAINER_READY_OK=true`, `LIVE_PG20_SEED_OK=true`, `LIVE_PG20_SUPPORT_OK=true`, `LIVE_PG20_COLUMNS_RICH_OK=true`, `LIVE_PG20_COMPOSITE_PK_OK=true`, `LIVE_PG20_FK_COMPOSITE_OK=true`, `LIVE_PG20_ROWS_FROM_SQL_SHAPE_OK=true`, `LIVE_PG20_HISTORICAL_UNNEST_NEGATIVE_OK=true`, `LIVE_PG20_INDEXES_OK=true`, `LIVE_PG20_MV_INDEXES_OK=true`, `LIVE_PG20_VIEW_NO_INDEXES_OK=true`, `LIVE_PG20_SEQUENCE_OK=true`, `LIVE_PG20_MULTI_SCHEMA_OK=true`, `LIVE_PG20_SCHEMA_SCOPE_OK=true`, `LIVE_PG20_SNAPSHOT_OK=true`, `LIVE_PG20_LOCKED_HELPERS_UNTOUCHED_OK=true`, `LIVE_PG20_STRICT_MARKER_COUNT=18`.

Rollup gate: `PHASE20_ALL_PASS=true`.

Non-strict diagnostics: `LIVE_PG20_DETECT_MS`, `LIVE_PG20_CONTAINER_MS`, `LIVE_PG20_SEED_MS`, `LIVE_PG20_SUITE_DURATION_S`, `LIVE_PG20_WALL_CLOCK_BUDGET_S=180`, `LIVE_PG20_WALL_CLOCK_OK=<true|false>`.
</strict_markers>

<threat_model>
The live lane is environment-gated. Main risks are runtime variance, slow image pulls, false skips, bootstrap matrix auto-enrollment, nondeterministic catalog ordering, broken marker accounting, and accidental helper/prod-SQL churn. The plan mitigates these by health-checking podman/docker before test launch, passing the selected provider into Go, using a build tag, failing CI through `LIVE_PG20_REQUIRED=1`, sorting canonical snapshot data, pinning PG16 by digest, adding shape preflight, and making the rollup fail closed.
</threat_model>
