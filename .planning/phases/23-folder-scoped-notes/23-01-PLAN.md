# 23-01 - Folder Lookup And Picker Namespace

<objective>
Add source-owned folder lookup for a connection and make the notes picker read/create "Global notes" from the active connection's folder namespace instead of legacy `global`.
</objective>

<threat_model>
- Risk: a manually edited folder sidecar could contain an unsafe folder ID. Mitigation in Wave 2 path helper validates folder IDs before filesystem operations.
- Risk: querying drawer cache would make picker correctness depend on drawer render state. Mitigation: Source-owned lookup and Handler facade only.
- Risk: multi-folder duplicate sidecar entries could expose multiple global note sets. Mitigation: keep FOLDER15 normalized single-folder behavior; no union.
</threat_model>

<must_haves>
- Do not add Go RPC endpoints or `vim.fn.Dbee*` calls.
- Do not read drawer `_folders_cache` directly.
- Keep local notes on `namespace_id = conn_id`.
- Keep hint row as `items[1]`.
- No `[global]` or `[local: ...]` tags in picker rows.
- No picker path may fall back to `namespace_get_notes("global")` for active global notes.
</must_haves>

<action>
Extend the folder Source API and Handler facade.

Required source behavior:

1. Add optional Source annotation:
   `get_folder_for_connection? fun(self: Source, conn_id: connection_id):Folder|nil`.
   Source implementations MUST return at most one folder per `conn_id`. Returning a list or multiple folders is a Source contract violation.
2. Implement `sources.FileSource:get_folder_for_connection(conn_id)`.
3. The method must call `self:load_folders()` so it honors `_ensure_folders_loaded()`, `_require_folders_writeable` safety, and `_normalize_folders()`.
4. Folder load parse/read failures must surface as `_folders_load_state == "load_failed"` or an explicit error return; do not silently collapse a corrupt sidecar to `{}` for Phase 23 authority checks.
5. Return the first normalized folder whose `connection_ids` contains `conn_id`; return nil when ungrouped. If folders cannot load, return `nil, error_kind`.
6. Do not scan raw JSON separately and do not implement union behavior.

Required handler behavior:

1. Add a Lua-only facade `Handler:get_folder_for_connection(conn_id)`.
2. It must locate the owning source from existing source connection lookups, then call the source method.
3. Return a table containing at least `source_id`, `folder_id`, and `folder_name`, or nil if no folder.
4. It must not call any `vim.fn.Dbee*` RPC endpoint except existing source connection helpers already used by Handler.
5. It must not depend on drawer UI state.
6. The Handler facade validates the Source contract defensively. If a Source returns a list or multiple folders, emit an ERROR-level notification, treat the lookup as nil, and fail closed. Do not keep first or create union semantics.
7. Add `Handler:list_all_folder_ids_across_sources()` or equivalent internal helper. It enumerates folder IDs from every folder-capable source and returns `(counts_by_folder_id, error_kind?)` so runtime duplicate-ID checks do not reimplement source traversal in UI modules. `error_kind` is set if any folder-capable source has `_folders_load_state == "load_failed"` or a capability call cannot prove folder IDs. The helper recomputes per call with NO caching at this layer. `_ensure_folders_loaded()` already memoizes per-source folder reads, so the cost is bounded to in-memory hash counts. If a future hot-path consumer needs caching, promote to a `lua/dbee/folder_authority.lua` single-source helper following the Phase 14 pattern with an `ALL_CONSUMERS_ROUTED` sentinel and explicit invalidation events; v1.5 backlog tracks this only if the helper enters a redraw or per-keystroke path.
8. `Handler:get_folder_for_connection(conn_id)` checks the returned folder ID against the cross-source counts. If `error_kind` is set, return `nil, error_kind` and let the caller show an empty global section with a folder-load-failure notification. If that folder ID appears in more than one source, emit `vim.notify('dbee: folder_id <id> exists in multiple sources; manual sidecar repair needed', vim.log.levels.ERROR)` and return nil.
9. Add `Handler:get_folder_metadata(folder_id)` or equivalent helper for display-only lookup by folder ID. It returns one unambiguous `{ source_id, folder_id, folder_name }` table or nil if missing/duplicated.
</action>

Change picker section construction to resolve the active folder namespace.

Required `NotePickerSections` shape:

1. Preserve existing fields:
   - `current_connection`
   - `global_notes`
   - `local_notes`
2. Add fields:
   - `global_namespace_id` string or nil
   - `current_folder` table or nil, with `id`, `name`, and `source_id`
3. For a current connection with a folder:
   - `global_namespace_id = notes_namespace.folder_namespace_id(folder_id)` (exact raw string remains `"folder:" .. folder_id`)
   - `global_notes = editor:namespace_get_notes(global_namespace_id)`
4. For a current connection without a folder:
   - `global_namespace_id = nil`
   - `global_notes = {}`
5. For no current connection:
   - `global_namespace_id = nil`
   - `global_notes = {}`
6. Preserve local notes:
   - `sections.local_notes = editor:namespace_get_notes(conn_id)` when a current connection exists.
7. Update `editor_get_all_notes()`:
   - `editor_get_all_notes()` reads ONLY the active folder namespace and the current connection's local namespace.
   - It does NOT call `namespace_get_notes("global")` under any condition.
   - There is no "pre-migration test mode" branch.
   - Tests requiring pre-migration state must construct fixtures with the legacy `global` directory present BUT the migration sentinel ALSO pre-written, then assert post-migration semantics.
7a. Enumerate every caller of `editor_get_all_notes()` BEFORE this change. Required callers list, verified by grep at plan-execute time:
   - history-search picker in `lua/dbee.lua`
   - possible external API consumers via `dbee.api.ui.editor_get_all_notes()`
   For each caller, confirm the new shape (folder + local only, no global) is correct.
7b. Namespace values emitted for folder-scoped notes are locked: `namespace = notes_namespace.folder_namespace_id(folder_id)` (raw namespace string exactly `"folder:" .. folder_id`). Tests assert exact string match.
   Update the note row namespace field annotation to accept raw namespace strings (`"folder:<folder_id>"` and connection IDs). Do not leave an annotation that implies only legacy `"global"`/local values.
7c. Add `editor_get_notes_for_connection(conn_id)` API. History-row picker passes `item.entry.conn_id` from the selected history row, NOT the active connection. This preserves history search across past connections.
   - Return shape is `{ id, name, namespace, file?, bufnr? }[]`.
   - `namespace` is the raw namespace string. Folder notes use `namespace = notes_namespace.folder_namespace_id(folder_id)`; local notes use `namespace = conn_id`.
   - No row has `namespace == "global"` under any condition.
   - Caller display formatting is separate from returned namespace data. History search in `dbee.lua` formats folder namespaces by looking up folder metadata via `Handler:get_folder_metadata(folder_id)` or equivalent and displaying the folder name; it must not rewrite the raw namespace field.
8. Add the single folder namespace helper module `lua/dbee/notes_namespace.lua`:
   - `validate_folder_id(folder_id) -> bool`
   - Accept ONLY `folder_id:match("^folder_[A-Za-z0-9]+$") ~= nil`
   - Every code path that constructs a `folder:<id>` namespace string MUST route through the validated helper.
   - Specifically, folder reads use `notes_namespace.folder_namespace_id(folder_id)` before `editor:namespace_get_notes(namespace)`, folder note creates use `notes_namespace.create_note_in_folder(editor, notes_dir, handler, folder_id, note_id, content)`, and folder note removes use the validated namespace helper before `editor:namespace_remove_note(namespace, note_id)`.
   - `notes_namespace.create_note_in_folder(...)` validates `folder_id`, calls `ensure_folder_namespace(notes_dir, folder_id, handler)` to apply collision/load-uncertainty authority, then calls `EditorUI:namespace_create_note(namespace, note_id, content, { from_authority = true })`.
   - Add a runtime guard in `EditorUI:namespace_create_note(namespace, ...)`: if `namespace` starts with `"folder:"`, throw unless the authority flag is present. Direct `namespace_create_note()` calls remain allowed for non-folder namespaces such as local connection IDs.
   - Invalid `folder_id` rejects with a descriptive error before filesystem access.
   - `lua/dbee/notes_namespace.lua` is the canonical location. Do not define `validate_folder_id` inline in `EditorUI` or any other module.
</action>
