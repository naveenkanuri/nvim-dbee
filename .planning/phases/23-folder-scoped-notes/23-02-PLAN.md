# 23-02 - Migration And Folder Namespace Lifecycle

<objective>
Migrate legacy global notes into folder namespaces once, prevent `global` recreation, and make folder create/delete/rename/move operations maintain note namespace semantics.
</objective>

<threat_model>
- Risk: concurrent nvim instances both migrate and delete/copy the same files. Mitigation: atomic lock directory plus post-lock sentinel recheck.
- Risk: folder deletion permanently deletes notes before folder removal fails. Mitigation: stage namespace directory and restore on handler failure.
- Risk: unsafe folder IDs from manual sidecar edits escape notes root. Mitigation: validate folder IDs before namespace filesystem operations.
- Risk: migration partial failure deletes legacy global. Mitigation: clone first, backup/rename second, sentinel last.
- Risk: bind mounts or symlinks make rename cross filesystem. Mitigation: same-filesystem preflight and fail-closed `EXDEV` handling; no copy+unlink fallback.
- Risk: stale staging/trash dirs leak deleted or partially migrated notes after a crash. Mitigation: startup GC of stale version-coupled scratch dirs before lock acquisition.
</threat_model>

<must_haves>
- Migration is first-launch post-Phase-23 and idempotent.
- Migration clones, not moves, each legacy global note into every existing folder namespace.
- Original `notes/global/` path is removed after successful clone/backup.
- Backup remains available until manual cleanup.
- Backup path is canonical: `notes/global.bak`, or `notes/global.bak.YYYYMMDDHHMMSS` only if the default backup already exists.
- Folder created after sentinel gets an empty namespace only.
- Folder rename never moves a namespace directory.
- Folder delete confirmation mentions note deletion.
- Editor startup never calls `namespace_create_note("global", "welcome")` after Phase 23 migration.
</must_haves>

Required module boundaries:

1. `lua/dbee/notes_namespace.lua` is the canonical namespace/lifecycle module. It exports:
   - `validate_folder_id(folder_id) -> bool`
   - `folder_namespace_id(folder_id) -> string`, exactly `"folder:" .. folder_id`
   - `folder_namespace_dir(notes_dir, folder_id) -> string`, exactly `notes_dir .. "/" .. folder_namespace_id(folder_id)`

   - `delete_folder_namespace(notes_dir, folder_id, source_id, handler) -> boolean, err?`
   - `list_existing_folder_namespaces(notes_dir) -> string[]`
2. Validate folder IDs before filesystem use. Accept ONLY when `folder_id:match("^folder_[A-Za-z0-9]+$")` is non-nil. The regex IS the gate. NO additional substring blacklist. Reject all other inputs including: empty string, nil, `folder` (no underscore-anything), `Folder_abc` (case-sensitive), `folder_a/b`, `folder_a%00b` (NUL byte), `folder_a.b`, `..`, `folder_-x`.
3. `ensure_folder_namespace(notes_dir, folder_id, handler?)` calls the Handler cross-source count helper when `handler` is provided. If `folder_id` exists in more than one source, refuse with `vim.notify('dbee: folder_id <id> exists in multiple sources; manual sidecar repair needed', vim.log.levels.ERROR)` before filesystem access.
4. `EditorUI` may expose thin wrappers for configured directory access, but it must delegate validation, ensure, delete, and list behavior to `notes_namespace`. Migration imports `notes_namespace` directly and does not instantiate `EditorUI`.
5. `lua/dbee/notes_migration.lua` owns migration orchestration and imports `notes_namespace`.

Required migration:

0. Before lock acquisition, scan `notes_dir` for `.notes-migration-v1.staging-<pid>-*` and `.notes-migration-v1.trash-<pid>-*` dirs. For each, parse pid and stat mtime. If pid is no longer alive (`vim.loop.kill(pid, 0)` returns false) OR mtime age is greater than 1 hour, remove it with `vim.loop.fs_rmrf`/equivalent. Emit the count of GC'd dirs to stderr or a debug log.
1. Migration runs once per session via `M.maybe_run_notes_migration(handler)` in `lua/dbee/notes_migration.lua`. Invoke it at the END of `setup_handler()` in `lua/dbee/api/state.lua:59-68`, AFTER `Handler:new()` returns and BEFORE `setup_handler()` returns to its first caller. Use `handler:get_sources()` to enumerate sources; `handler.sources` is a map, not an ipairs array.
2. Inside `maybe_run_notes_migration(handler)`:
   - set a once-per-process flag `m.migration_attempted = true` before running body to prevent re-entry on later `setup_handler()` calls.
   - force-load folders on every folder-capable source: for each source from `handler:get_sources()`, if `supports_folders()` and `load_folders` exist, call `src:load_folders()` to synchronously trigger `_ensure_folders_loaded()`.
   - after loads, validate every folder-capable source has `_folders_load_state == "loaded_ok"`. Any `load_failed` aborts with `vim.notify('dbee: migration aborted — folder source load failed; fix sidecar before retrying', vim.log.levels.WARN)` and no sentinel write.
   - empty folder list under `loaded_ok` proceeds normally per step 5a.
3. Use sentinel path `notes_dir .. "/.notes-migration-v1"`.
4. Use lock directory path `notes_dir .. "/.notes-migration-v1.lock"` and acquire it FIRST with `vim.loop.fs_mkdir(path, 448)`.
   - If mkdir returns EEXIST, `vim.loop.fs_stat(lock_path)` and compute `age_seconds = os.time() - stat.mtime.sec`.
   - If `age_seconds < 0`, treat as fresh lock and fail fast with concurrent-instance message.
   - If `age_seconds < 300`, treat as fresh lock and fail fast with concurrent-instance message.
   - If `age_seconds >= 300`, attempt `vim.loop.fs_rmdir(lock_path)` and retry mkdir once.
4a. Before any staging mkdir, call `vim.loop.fs_statfs(notes_dir)` and `vim.loop.fs_statfs(notes_dir .. '/.notes-migration-v1.staging-precheck-tmp')` after creating the precheck dir as a sibling. Cleanup the precheck dir. If device/filesystem IDs differ, abort with `vim.notify('dbee: migration aborted — cross-filesystem rename detected; not supported. Move notes/ off bind mounts and retry.')`.
5. After acquiring lock, re-check sentinel. If `notes/.notes-migration-v1.recovery-needed` exists from a previous sentinel-write failure, read and validate the JSON manifest before writing the missing sentinel. Do not clear the recovery manifest unless validation succeeds.
5a. AFTER lock acquisition AND AFTER sentinel re-check, enumerate folders via `notes_namespace.list_existing_folder_namespaces(notes_dir)` plus Handler/Source folder IDs. Folders created by other instances AFTER this enumeration are intentionally not seeded.
   - If total folder count is zero and `notes/global/` is absent or has zero readable `*.sql` files, write `.notes-migration-v1` and exit cleanly.
   - If total folder count is zero and `notes/global/` has readable `*.sql` files, create canonical backup, delete `notes/global/`, write `.notes-migration-v1`, and exit cleanly without clones. Legacy notes survive in backup for manual user relocation.
5b. Duplicate folder ID guard:
   - enumerate folder IDs across all folder-capable sources.
   - if any duplicate folder ID appears across sources, abort before namespace creation.
   - emit `vim.notify('dbee: duplicate folder_id detected across sources; migration aborted. Repair sidecars before retry.')`.
   - do NOT write sentinel, delete `notes/global/`, or create backup.
5c. If `notes/global/` exists:
   - collect readable `*.sql` files only.
   - emit `vim.notify('dbee: migrating <N> notes into <F> folders')` before clone.
   - stage all clones into `notes/.notes-migration-v1.staging-<pid>-<random>/folder:<id>/` directories.
   - NO writes go directly to final `notes/folder:<id>/` paths during clone.
   - clone each global SQL file into each staged folder namespace, preserving basename.
   - if a target filename already exists, preserve existing target and prepare a collision-safe suffix such as `<stem>.global-migrated.sql`.
5d. Verify all staged paths exist and match the expected file count. If verification fails, `rmrf` the staging dir and abort migration without sentinel/backup/global delete.
5e. Promote staging per file:
   - For each folder namespace, ensure final dir exists with `vim.loop.fs_mkdir` if missing.
   - For each staged file, compute final path. If final path exists, use `<stem>.global-migrated.sql` and add a numeric suffix if needed.
   - Rename staged file to final file with `vim.loop.fs_rename`.
   - Track every migration-created final file path in a manifest table.
   - If `fs_rename` returns `EXDEV`, abort with the GN-20 notification and do not attempt copy+unlink fallback.
   - On any non-EXDEV per-file rename failure, rollback by deleting only manifest-tracked files created by this migration. Do NOT delete the entire final namespace dir because it may contain user-created folder notes.
6. Backup AFTER successful promotion. Create backup by moving/copying legacy global dir only to `notes/global.bak`, or `notes/global.bak.YYYYMMDDHHMMSS` if `global.bak` already exists. If backup creation fails, log warning but continue; backup is for safety, not correctness after clones are promoted.
7. AFTER backup, remove the original `global/` path. If delete fails, log warning. Sentinel still gets written because clones are in place.
8. Write sentinel atomically by writing `<sentinel>.tmp` then renaming. If sentinel write fails, the migration is functionally complete but sentinel-driven idempotency is broken; emit warning and write `notes/.notes-migration-v1.recovery-needed` containing JSON:
   ```json
   {
     "folder_ids": ["folder_X", "folder_Y"],
     "clones_per_folder": 3,
     "clone_basenames": ["welcome.sql", "cleanup.sql", "truncate_individual.sql"],
     "backup_path": "notes/global.bak",
     "global_deleted": true,
     "timestamp": "<ISO-8601>"
   }
   ```
   Recovery on next launch runs post-lock and post-sentinel-recheck. It reads the manifest, verifies each `folder_ids[i]/clone_basenames[j]` file exists with non-zero size, writes `.notes-migration-v1`, then removes the manifest. If verification fails, log an error and retain the manifest for user intervention.
9. If `notes/global/` does not exist:
   - create current folder namespace dirs.
   - write sentinel.
10. Always release lock on success or failure.
11. Emit completion notification after successful migration.
