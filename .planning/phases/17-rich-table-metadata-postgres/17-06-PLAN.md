# 17-06 - PostgreSQL Strict Tests And Markers

<objective>
Add deterministic PostgreSQL rich metadata validation that mirrors the Oracle `RICH16_*` suite while keeping Docker/testcontainer coverage optional.
</objective>

<must_haves>
- New marker script uses `RICH_PG_*` names only.
- Existing `ci/headless/check_rich_metadata.lua` and `RICH16_*` markers remain unchanged unless a Wave 5 assertion needs shared helper-safe adjustment.
- Strict tests must not require Docker.
- `make perf-lsp` runs both Oracle and PostgreSQL rich marker scripts.
</must_haves>

<threat_model>
The risk is false confidence from static marker checks that do not cover parser behavior. This wave combines sqlmock Go tests for adapter parsing, behavioral headless Lua fixtures for drawer/singleflight/backpressure, and benchmark diagnostics for large metadata result sets.
</threat_model>
