# 17-04 - Msgpack And Event Payload Additive Fields

<objective>
Preserve the new PostgreSQL Column and Index fields through existing sync msgpack wrappers and async `structure_children_loaded` Lua payload serialization.
</objective>

<must_haves>
- No new RPC endpoints or event names.
- Extend existing `columnWrap`, `indexWrap`, `columnsToLua`, and `indexesToLua`.
- Empty strings should serialize as absent/nil where existing payload style supports it.
- Old payload tests must keep passing.
</must_haves>

<threat_model>
Compatibility risk is that msgpack or Lua event paths silently drop new fields, making drawer labels impossible. Regression risk is that required old fields change shape. This wave only adds fields to existing serializers.
</threat_model>
