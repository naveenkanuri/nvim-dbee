# 17-01 - Additive Go Core Metadata Fields

<objective>
Extend shared Go rich metadata structs so PostgreSQL can carry generated, identity, default, serial-sequence, and INCLUDE-index metadata without breaking old payloads or Oracle X.1.
</objective>

<must_haves>
- `core.Column` keeps `Name` and `Type` unchanged and adds only optional fields.
- `core.Index.Columns` continues to mean key columns only.
- `core.Index.IncludeColumns` is separate from key columns.
- All new fields use both JSON and msgpack `omitempty` tags.
</must_haves>

<threat_model>
Backward compatibility is the blocking risk. Old clients must decode payloads that omit new fields, and old-style `{name,type}` structs must ignore new JSON fields. No external input is introduced in this wave.
</threat_model>
