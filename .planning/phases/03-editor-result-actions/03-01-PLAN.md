1. Create `ci/headless/check_result_export.lua` following headless test pattern:
   - Instantiate a real-ish ResultUI stub that includes real `get_actions()` returning the actual export_result closure, backed by stub state: `current_call = { id = "test-id" }`, `total_rows = 42`, and a `handler` with a recording `call_store_result` stub.
   - Mock `vim.ui.input` to auto-provide a path.
   - Mock `vim.fn.filereadable` to return 0 (no overwrite prompt needed).
   - Test the 5 behaviors above by calling actions from `get_actions()`. For the overwrite path, mock `vim.ui.select` and `vim.fn.filereadable = 1`.

2. Add `export_result` to `ResultUI:get_actions()` in `lua/dbee/ui/result/init.lua`:
   - Guard: `if not self.current_call then utils.log("warn", "No results to export") return end`
   - IMPORTANT: Capture state BEFORE async prompts to avoid race conditions:
     ```lua
     local call_id = self.current_call.id
     local row_count = self.total_rows or 0
     ```
     Use `call_id` and `row_count` in all callbacks below (not `self.current_call.id` or `self.total_rows` which may change during user interaction).
   - Prompt: `vim.ui.input({ prompt = "Export to: ", default = vim.fn.getcwd() .. "/result.csv" }, function(path) ... end)`
   - Nil/empty path guard (user pressed Escape): `if not path or path == "" then return end`
   - Format inference: `local ext = vim.fn.fnamemodify(path, ":e"):lower()` with map `{ csv = "csv", json = "json" }`. If not in map: `utils.log("warn", "Unsupported format '." .. ext .. "'. Use .csv or .json")` and return.
   - Large export warning: `if row_count > 10000 then utils.log("info", string.format("Exporting %d rows - this may take a moment...", row_count)) end`
   - Overwrite guard: `if vim.fn.filereadable(path) == 1 then vim.ui.select({"No", "Yes"}, { prompt = "File exists. Overwrite?" }, function(choice) if choice == "Yes" then do_export() end end) else do_export() end`
   - `do_export()`: `pcall(self.handler.call_store_result, self.handler, call_id, format, "file", { extra_arg = path })`. On error: `utils.log("error", "Export failed: " .. tostring(err))`. On success: `utils.log("info", string.format("Exported %d rows to %s", row_count, path))` (per CONTEXT.md locked decision).
   - Note: `row_count` reflects the display-side count which may differ from actual exported rows for large result sets. This is a known cosmetic limitation — the Go backend exports all rows regardless.

3. Add default keybinding in `lua/dbee/config.lua` result.mappings section:
   - `{ key = "ge", mode = "n", action = "export_result" }` with comment "-- export results to file"
