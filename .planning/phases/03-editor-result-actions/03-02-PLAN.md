3. Add `dbee.explain_plan(opts)` public API function in `lua/dbee.lua`:
   - Place it near `dbee.execute_context()` for locality.
   - Pattern follows execute_context: ensure_core_available -> get_current_connection -> extract query -> dispatch.
   - Guard: no connection -> `utils.log("warn", "No connection selected. Select one from the drawer, then run again.")` and return.
   - Query extraction: Call `extract_query_from_context(conn, opts)` — handles both normal and visual mode.
   - Guard: empty query -> `utils.log("warn", "No SQL found at cursor. Place cursor on a query and try again.")` and return.
   - Known limitation: bind variables (`:var`) are NOT resolved for explain plan. This is intentional — most databases handle EXPLAIN structurally without actual values. Oracle may reject unresolved binds; document this as a known edge case.
   - Adapter dispatch table:
     ```lua
     local wrappers = {
       postgres = function(q) return "EXPLAIN " .. q end,
       mysql    = function(q) return "EXPLAIN " .. q end,
       sqlite   = function(q) return "EXPLAIN QUERY PLAN " .. q end,
     }
     ```
   - Oracle special case (CRITICAL — connection_execute is async, returns immediately):
     Use async event-driven chaining via a SINGLETON `call_state_changed` listener and a pending map keyed by step-1 call id. Do NOT register one listener per explain call (event bus has no unregister API, so per-call registration leaks callbacks and degrades performance over time).
     ```lua
     -- module-level state in dbee.lua
     local explain_oracle_pending = {} -- [step1_call_id] = { conn_id, timer }
     local explain_oracle_listener_registered = false

     local function stop_timeout_timer(timer)
       if not timer then return end
       pcall(function()
         if timer.stop then timer:stop() end
         if timer.close then timer:close() end
       end)
     end

     local function ensure_oracle_explain_listener()
       if explain_oracle_listener_registered then
         return
       end
       explain_oracle_listener_registered = true
       api.core.register_event_listener("call_state_changed", function(data)
         if not data or not data.call or not data.call.id then
           return
         end
         local pending = explain_oracle_pending[data.call.id]
         if not pending then
           return
         end
         local state = data.call.state
         if not terminal_states[state] then
           return -- keep timer running until terminal
         end

         stop_timeout_timer(pending.timer)
         explain_oracle_pending[data.call.id] = nil

         if state ~= "archived" then
           utils.log("warn", "Explain plan failed (step 1): " .. (data.call.error or state))
           return
         end

         local ok2, step2_call = pcall(api.core.connection_execute, pending.conn_id,
           "SELECT * FROM TABLE(DBMS_XPLAN.DISPLAY)")
         if not ok2 then
           utils.log("warn", "Explain plan failed (step 2): " .. tostring(step2_call))
           return
         end
         api.ui.result_set_call(step2_call)
         dbee.open()
       end)
     end

     if adapter == "oracle" then
       ensure_oracle_explain_listener()

       local step1 = "EXPLAIN PLAN FOR " .. query
       local ok1, step1_call = pcall(api.core.connection_execute, conn.id, step1)
       if not ok1 then
         utils.log("warn", "Explain plan failed: " .. tostring(step1_call))
         return
       end

       local step1_id = step1_call.id
       local timeout_timer = vim.defer_fn(function()
         explain_oracle_pending[step1_id] = nil
         utils.log("warn", "Explain plan timed out (step 1 took >10s)")
       end, 10000)
       explain_oracle_pending[step1_id] = {
         conn_id = conn.id,
         timer = timeout_timer,
       }
       return
     end
     ```
     Note: `terminal_states` table already exists in dbee.lua. With this design, listener count stays constant, pending state is cleaned deterministically, timer handles are canceled correctly (`:stop/:close`), and the UI remains non-blocking.
   - Unsupported adapter: `utils.log("warn", "Explain Plan not supported for " .. conn.type)` and return.
   - On success: `api.ui.result_set_call(call)` and `dbee.open()`.
