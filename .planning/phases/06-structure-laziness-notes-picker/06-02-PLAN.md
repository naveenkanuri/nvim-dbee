- Keep `dbee.pick_notes()` as the public command path.

- Local notes are defined only by `namespace_get_notes(tostring(current_connection.id))`. Do not reinterpret local membership from execution or reconnect metadata.
- Empty state stays explicit: no notes at all means info log and no picker; no current connection means only Global; globals-without-locals means show the Local section with a hint row.
</constraints>

<interfaces>
Preserve the existing public picker entry point:
```lua
function dbee.pick_notes()
  -- still opens one picker and on confirm calls:
  api.ui.editor_set_current_note(note_id)
end
```

Add a picker-specific structured helper:
```lua
---@class NotePickerSections
---@field current_connection { id: string, name: string }|nil
---@field global_notes note_details[]
---@field local_notes note_details[]

api.ui.editor_get_note_picker_sections() -> NotePickerSections
```

Recommended picker row shape:
```lua
---@class NotePickerItem
---@field kind "header"|"hint"|"note"
---@field note_id? string
---@field section "global"|"local"
---@field text string
---@field tag? string
---@field selectable boolean
```

Recommended ordering:
1. `Global notes` header when `global_notes` is non-empty
2. global note rows, if any
3. `Local notes (<conn_name>)` header when a current connection exists
4. local note rows, if any
5. local empty hint row when a current connection exists but `local_notes` is empty

If `global_notes` is empty and local notes exist, omit the empty Global header entirely and let the Local section render first.
</interfaces>

  <action>
    Upgrade the existing picker path without breaking the flat helper contract.

    1. Add a new structured helper in `api.ui`, for example `editor_get_note_picker_sections()`, that:
       - fetches global notes from `namespace_get_notes("global")`
       - fetches local notes from `namespace_get_notes(tostring(current_connection.id))` when a current connection exists
       - returns current connection display name for header/tag rendering
       - leaves `editor_get_all_notes()` untouched for unrelated callers

    2. Refactor `dbee.pick_notes()` to consume the new structured helper and build one picker with pseudo-header/hint rows plus note rows.
       - call `editor_get_note_picker_sections()` exactly once per picker open
       - build all picker rows from that returned snapshot
       - confirm callbacks operate only on the prepared row metadata; they MUST NOT refetch note sections or namespaces

    3. Use fixed ordering per D-41:
       - Global section first
       - Local section second when a current connection exists

    4. Tag selectable note rows clearly:
       - global rows carry `[global]`
       - local rows carry `[local: <conn_name>]`

    5. Keep empty-state behavior explicit:
       - no notes at all -> `utils.log("info", "No notes found")`, no picker
       - no current connection -> no Local section
       - current connection present but no local notes -> header plus non-selectable hint row
       - global notes empty but local notes present -> omit the empty Global header entirely

    6. Guard selection:
       - only `kind = "note"` rows are confirmable
       - the confirm callback MUST begin with `if not item or item.kind ~= "note" or not item.note_id then`
       - that guard MUST warn (`utils.log("warn", "Select a note row")` or equivalent), MUST NOT call `editor_set_current_note()`, MUST NOT call `picker:close()`, and MUST return early
       - pseudo-header rows, hint rows, nil-item confirm payloads, and malformed `kind = "note"` rows without `note_id` must therefore remain visible/no-op safe after a forced confirm attempt

    7. Update `lua/dbee/doc.lua` if Phase 6 public docs describe the new helper or picker behavior.
  </action>
