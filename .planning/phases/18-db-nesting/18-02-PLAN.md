# 18-02 - Current Database Reporting Fixes

<objective>
Ensure eligible database-switching adapters can report the current database even when there are no available switch targets, so Wave 1 can render the active database node for single-DB connections.
</objective>

<must_haves>
- Modify only `dbee/adapters/postgres_driver.go`, `dbee/adapters/sqlserver_driver.go`, `dbee/adapters/redshift_driver.go`, and `dbee/adapters/list_databases_test.go`.
- Do not change handler APIs, event names, or drawer code in this wave.
- Keep `ListDatabases()` signatures unchanged.
- Fix only adapters measured in context as needing current-when-no-alternatives behavior.
- Preserve existing available-database behavior when alternatives exist.
- Add focused sqlmock tests for each fixed adapter. Broad `go test ./adapters` alone is not sufficient acceptance.
</must_haves>

<tasks>
<task id="18-02-T1" type="execute">
<read_first>
- `dbee/adapters/postgres_driver.go:125-149`
- `dbee/core/connection.go:93-97`
- `dbee/core/connection.go:254-266`
</read_first>
<action>
Update PostgreSQL `ListDatabases()` so it always returns `current_database()` even when the query for alternative databases returns no rows.

Acceptable implementation patterns:

1. Preferred: query current database first, then query available alternatives with the existing filter.
2. Alternative: use a SQL shape that always returns one current row and optional alternative rows, with nil-safe parsing.

If using the single-query shape, ensure the available column can be NULL and the Go parser skips nil/empty alternatives without losing current.

Add `TestPostgresListDatabasesNoAlternatives` in `dbee/adapters/list_databases_test.go`:

1. Construct a `postgresDriver` with `sqlmock` and `builders.NewClient(db)`.
2. Mock the no-alternatives path: the current query returns `dbee_test`; the alternatives query returns zero rows, or the single-query form returns only the current/NULL row.
3. Assert `current == "dbee_test"`, `available` is empty, and `err == nil`.
4. Add an alternatives-present regression case asserting non-current database names are still appended.
</action>
<verify>
```bash
env GOCACHE=/tmp/codex-go-cache go -C dbee test ./adapters -run TestPostgresListDatabases -v
env GOCACHE=/tmp/codex-go-cache go -C dbee test ./adapters
```
</verify>
<acceptance_criteria>
- `ListDatabases()` returns `current != ""` when no non-current database rows exist.
- Existing behavior still appends non-current database names to `available`.
- `TestPostgresListDatabasesNoAlternatives` fails if the alternatives query returns zero rows and `current == ""`.
- No named parameters or unsupported driver features are introduced.
</acceptance_criteria>
</task>
