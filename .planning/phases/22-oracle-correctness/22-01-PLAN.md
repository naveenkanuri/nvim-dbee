<threat_model>
Rejecting unsafe bind names is intentionally behavior-changing for queries that previously relied on reserved/problematic names, but those names are the class that can fail at the Oracle parser. The mitigation is a clear backend validation error with the offending bind name and a `p_` rename hint. Lua-side pre-validation is out of scope; reserved bind names surface as backend errors after the user-input prompt completes, and v1.5 may add a Lua mirror. The plan avoids the higher-risk alternative of rewriting SQL or bind maps. Source-scanning tests reduce future regression risk without live DB dependencies.
</threat_model>
