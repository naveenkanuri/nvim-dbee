# 24-01 - Oracle Bind Transparent Rewriter

<objective>
Accept Oracle-legal user bind names containing `$` or `#`, rewrite only driver-facing SQL placeholders and `sql.Named` keys to go-ora-compatible names, reverse-map errors to original names, and preserve all Phase 22/22.5 reserved-name and audit contracts.
</objective>

</files>

<must_haves>
- No live Oracle or testcontainers dependency.
- No new external Go dependency.
- No Lua files touched.
- No edits to locked helpers.
- Reserved/unsafe bind names still reject at user surface.
- User names containing `_x24_` or `_x23_` reject before rewrite.
- Sentinel and driver-name collision checks are case-insensitive.
- Internal static `p_` binds stay unchanged.
- Rewriter skips strings, q-quotes, quoted identifiers, line comments, and block comments.
- Q-quote support is narrow + fail-loud for unsupported single-quote and multibyte delimiters.

- `oracleSafeBindSuggestion` remains driver-valid after validator split.

- Tokenizer runs at most once per query/exec call and emits a shared `[]codeRegion` span list.
- Per-query rewrite path uses manual byte scanning, not regexp.
- Zero-scan fast path is disabled by q-quote candidates and cursor-marker candidates.
- SQL fast-path probes are allocation-free ASCII byte scans; no `strings.ToUpper(query)`.
</must_haves>

<action>
Create `dbee/adapters/oracle_bind_rewrite.go` with the tokenizer, name rewriter, and error reverse mapper.

Required helper shape:

1. `rewriteOracleBindNameForDriver(name string) (driverName string, changed bool, err error)`.
2. `scanOracleSQLCodeRegions(query string) ([]codeRegion, error)` or equivalent unexported helper.
   - This is the single lexical authority for bind rewrite, cursor detection, sentinel validation, and marker cleanup.
   - It runs at most once per query/exec call; downstream helpers consume the returned span list instead of rescanning.
   - The tokenizer is private to Oracle bind rewrite for Phase 24; future consumers must extract it by refactor rather than copying scanner logic.
3. `transformOracleSQLBinds(query string, regions []codeRegion, cursorMarkers []oracleCursorMarker) (rewritten string, mapping oracleBindRewriteMap, err error)` or equivalent single-pass output transform.
   - This transform emits output from the original SQL once.
   - It strips code-region cursor marker comments and rewrites `$` / `#` bind refs while walking original spans.
   - It preserves regular comments and skipped-region marker-shaped text.
   - It does not mutate SQL and then reuse stale span offsets.
4. The mapping tracks at least:
   - original user name
   - rewritten driver name
   - `driverToUser map[string]string`
   - `userToDriver map[string]string`
5. Rewriter only changes bind references in code regions matched by `:([A-Za-z_][A-Za-z0-9_$#]*)`.
6. Rewriter skips:
   - single-quoted strings with doubled quote escapes
   - q-quoted strings using the narrow + fail-loud Phase 24 delimiter rule:
     - `q'X...X'` / `Q'X...X'` where opener `X` is a single ASCII byte that is not whitespace and not `'`
     - if opener is `(`, `{`, `[`, or `<`, terminator is the matched closer `)`, `}`, `]`, or `>` followed by `'`
     - otherwise terminator is the same opener `X` followed by `'`
     - whitespace, single-quote, and multibyte UTF-8 openers reject as unsupported tokenizer errors
     - unsupported delimiter tests emit diagnostic `LIVE_PG24_QQUOTE_UNSUPPORTED=true`; full support is deferred to v1.5
   - double-quoted identifiers
   - line comments
   - block comments
7. `:=` must not match because `=` is not a valid first bind-name character.
8. The tokenizer and bind scanner are manual byte-scan code. Do not use regexp in the per-query rewrite path.
9. Case-folding for extracted bind-name collision checks may use `strings.ToUpper` on short ASCII bind names; SQL text probes must use allocation-free ASCII byte loops.
10. Add allocation-free SQL probe helpers:
   - `containsOracleSentinelASCII(query)` scans byte-by-byte for `_x24_` / `_x23_` case-insensitively using ASCII folding such as `(b | 0x20)`.
   - `containsOracleQQuoteCandidateASCII(query)` first checks `strings.IndexAny(query, "qQ")`, then lookaheads for `q'` / `Q'`.
   - `containsOracleCursorMarkerCandidateASCII(query)` checks for `/*` plus ASCII case-insensitive `CURSOR`.
11. Use a 5-step no-change fast path:
   - Step 0: if `containsOracleQQuoteCandidateASCII(query)` is true, run tokenizer + ORA24-25 fail-loud/parity handling even when there are no binds.
   - Step 0.5: if `containsOracleCursorMarkerCandidateASCII(query)` is true, run the code-region scanner and cursor-marker scan; route to REF CURSOR path when a code-region marker is found even if no `$` / `#` rewrite is needed.
   - Step 1: if `len(bindMap) == 0` and the SQL text contains no `$`, `#`, sentinel, q-quote candidate, or cursor-marker candidate under allocation-free probes, return immediately with zero allocations.
   - Step 2: otherwise, if `strings.IndexAny(query, "$#") == -1`, SQL allocation-free probes find no sentinel/q-quote/cursor candidate, and bind-map key scan finds no `$`, `#`, `_X24_`, or `_X23_`, return immediately with zero allocations.
   - Step 3: otherwise, run the full tokenizer/transform path.
   - A `$` / `#` only inside comments or strings may pessimize into scanning; correctness wins over optimizing this edge.
   - Allocate `strings.Builder` only after the first rewrite is required, pre-sized to `len(query)`.
   - Pre-size maps to the number of rewritten/validated binds when known, otherwise to bind-map size.
12. If SQL text contains `_X24_` / `_X23_`, run code-region validation. Sentinel text inside skipped strings/comments is ignored, but code-region bind refs such as `:my_x24_1` reject with the locked sentinel-collision error.
13. Detect case-insensitive driver-name collisions while building the rewrite map. If multiple original user names uppercase-fold to the same rewritten driver name, reject with a case-insensitive collision error.
14. Error reverse mapper handles colon-prefixed driver bind occurrences and known bare go-ora parameter templates with originals and wraps with `Unwrap() error`; see T5.
</action>
