# 21-01 - Column Completion Annotation Rendering

<objective>
Pre-bake rich metadata annotations into cached column completion items while keeping `label` and `insertText` equal to the raw column name.
</objective>

<must_haves>
- Keep `item.label = col.name` and `item.insertText = col.name`.
- Set `item.labelDetails.detail` for modern LSP 3.17 clients.
- Set `item.detail` for older clients that ignore `labelDetails`.
- Do not add default-expression popup markers in this wave.
- Do not change disk cache version or serialized column payloads.
- Annotation strings (`labelDetails.detail`, hybrid `detail`, reverse-FK doc text) are derived only into in-memory completion/doc structures. They MUST NOT be written onto `Column` records or into `_save_columns_to_disk`; the v4 serialized payload remains `{ version, schema_filter_signature, root_epoch, columns = Column[] }`.
- Marker construction must be append-only (`markers[#markers + 1] = marker`); fixed-index marker assignment is forbidden because `#markers == 0` relies on no nil holes.
- Do not edit locked helpers.
</must_haves>

<action>
Add local annotation helpers in `schema_cache.lua` near the completion item helpers:

1. `rich_fk_target_columns(ref)`:
   - Prefer `ref.target_columns` when it is a non-empty list.
   - Fall back to `{ ref.target_column }`.
   - Never render both the shorthand and array values.
2. `rich_fk_marker(ref)`:
   - Skip malformed refs with no target table or no target column(s).
   - Single target: `[FK→target_table.target_col]`.
   - Composite target: `[FK→target_table.(a,b)]`.
   - Omit target schema from popup markers to keep the completion row compact.
3. `column_annotation_markers(col, pk_count)`:
   - construct `markers` only with append operations:
     ```lua
     if pk_marker then markers[#markers + 1] = pk_marker end
     ```
   - never assign markers by fixed numeric index such as `markers[1] = ...`; nil holes make Lua's `#markers` boundary undefined.
   - PK:
     - compute `pk_count := count(c in cols where c.primary_key == true)` once per table;
     - if `pk_count == 1` and `col.primary_key == true`, render `[PK]`;
     - if `pk_count >= 2` and `col.primary_key_ordinal > 0`, render `[PKn]`;
     - if `pk_count >= 2` and `col.primary_key_ordinal` is `0`/nil, render `[PK]` with no fabricated ordinal.
   - FK:
     - iterate `col.foreign_keys` only if it is a list;
     - render all distinct valid FK markers, space-joined;
     - dedupe by constraint name + target table + target columns.
   - Nullable:
     - append `null` only when `col.nullable == true` and the column is not PK.
     - omit when nullable is false or nil.
   - Unsupported rich metadata adapters therefore produce no annotation text.
4. `column_completion_detail(col, markers)`:
   - return `col.type` when `#markers == 0`;
   - otherwise return `col.type .. " " .. table.concat(markers, " ")`.
</action>

<action>
Update `_update_column_index(schema, table_name)` to use the helpers.

Required shape:

```lua
local markers = column_annotation_markers(col, pk_count)
local has_markers = #markers > 0
local markers_text = has_markers and table.concat(markers, " ") or nil

{
  label = col.name,
  kind = CompletionItemKind.Field,
  labelDetails = has_markers and { detail = " " .. markers_text } or nil,
  detail = column_completion_detail(col, markers),
  insertText = col.name,
  sortText = "0_" .. col.name,
}
```

Implementation details:

1. Precompute `pk_count` from the table's columns before building items.
2. Keep `label` raw because `_copy_items_with_data` uses `item.label` to populate `data.column`.
3. Keep `sortText` stable.
4. Fast-path predicate is `#markers == 0`; do not compute `markers_text` for empty-marker columns. The common no-rich-marker path must set `labelDetails = nil`, not `{}`.
5. Do not mutate returned items at read time; `get_column_completion_items` already returns deep copies.
6. Do not touch `server.lua` unless an implementation-time test proves the server strips `labelDetails`.
7. Keep the wide-table hot path within the Wave 5 budget: 500-column table completion p95 `< 2.5ms` and `<= 5us/column`.
</action>
