# 21-05 - Performance Gates

<objective>
Lock performance budgets for completion-item annotation reads, reverse-FK index construction, and resolve-time reverse-FK lookups.
</objective>

<must_haves>
- Perf tests run in headless Neovim and do not require live DBs.
- Fixtures are deterministic and do not depend on wall-clock sleeps.
- Request paths remain cache-only.
- Perf markers are strict because Phase 21 affects per-keystroke completion UX.
</must_haves>

<action>
Create `ci/headless/check_lsp21_perf.lua`.

Fixture:

1. Build 1,000 tables x 10 columns = 10,000 columns.
2. Include 1,000 FK refs distributed across those tables.
3. Include at least 100 composite FK groups.
4. Warm the cache before measurement.

Measurements:

1. Completion read p95:
   - call `get_column_completion_items` for hot tables at least 200 times;
   - p95 must be `< 1ms`.
2. Reverse-index build:
   - time `_rebuild_column_indexes()` or equivalent public test hook around the 10k/1k-FK fixture;
   - total must be `< 50ms`.
3. Resolve lookup p95:
   - call `get_reverse_fk_refs` at least 200 times for FK-target and non-target columns;
   - p95 must be `< 1ms`.
4. Emit diagnostic numeric markers for measured p95/build values if useful, but keep only the three strict perf markers counted.
</action>
