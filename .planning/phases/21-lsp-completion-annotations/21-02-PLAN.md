# 21-02 - Reverse-FK Index

<objective>
Maintain an in-memory reverse-FK index derived from loaded column metadata and expose a fail-closed cache API for resolve-time lookup.
</objective>

<must_haves>
- The index is derived from `self.columns`; it is never written to disk.
- Reads route through `epoch_authority.read_with_freshness`.
- Rebuild and clear through existing column-index lifecycle.
- Maintain both target and source maps so source-table eviction is O(refs-for-source), not O(total reverse refs).
- Do not scan all tables on every resolve hover.
- Do not edit `epoch_authority.lua`.
</must_haves>

<action>
Add in-memory reverse-FK state to `SchemaCache:new` and reset paths.

State:

```lua
reverse_fk_refs_by_target_key = {},
reverse_fk_refs_by_source_key = {},
reverse_fk_index_epoch = 0,
reverse_fk_index_size = 0,
reverse_fk_index_overflow = false,
reverse_fk_overflow_warned = false,
```

Caps:

```lua
REVERSE_FK_PER_TARGET_REF_CAP = 50
REVERSE_FK_PER_SOURCE_REF_CAP = 100
REVERSE_FK_TOTAL_REF_CAP = 50000
```

Reset requirements:

1. `_reset_indexes()` clears reverse-FK state.
2. `_drop_column_index(key)` calls `_drop_reverse_fk_refs_for_source(key)`.
3. `_drop_reverse_fk_refs_for_source(source_key)` uses `reverse_fk_refs_by_source_key[source_key]` to remove only target entries contributed by that source table. It must not walk the entire `reverse_fk_refs_by_target_key` map.
4. `_rebuild_column_indexes()` clears reverse-FK state before looping all cached columns, then rebuilds column completion items, target map, and source map.
5. After `_rebuild_column_indexes()` completes a successful rebuild, set `self.reverse_fk_index_epoch = epoch_authority.cache_epoch(self)`.
6. After `_refresh_reverse_fk_refs_for_table(...)` mutates entries inside admitted `_store_columns`, set `self.reverse_fk_index_epoch = epoch_authority.cache_epoch(self)`.
7. `_reset_indexes()` and fail-closed clear paths set `reverse_fk_index_epoch = 0`.
8. `invalidate()` and `set_connection()` already call `_reset_indexes`; no extra invalidation branch is needed.
</action>

<action>
Add reverse-FK index writers.

Required helpers:

1. `reverse_fk_target_key(schema, table, column)`:
   - logical shape is `target_schema:target_table:target_column`;
   - implementation may use a safe delimiter such as `"\31"` to avoid collisions.
   - writer and reader both normalize each component with `schema_name_canonical.canonical(name, quoted=true, self.fold_id).canonical` before joining;
   - display fields keep the exact adapter-provided schema/table/column text.
2. `reverse_fk_source_key(schema, table)` returns the existing `table_key(schema, table)` source identity and is the key in `reverse_fk_refs_by_source_key`.
3. `reverse_fk_pair_from_ref(ref)`:
   - precedence 1: if `source_columns`/`target_columns` are arrays and `source_ordinal > 0` is in bounds for both arrays, pair `SourceColumns[source_ordinal]` with `TargetColumns[source_ordinal]`;
   - precedence 2: if `source_column` exactly appears in `source_columns`, use that index to select `target_columns[index]`;
   - precedence 3: if both `source_column` and `target_column` are set, use that shorthand pair as a single-column FK;
   - precedence 4: skip malformed refs and log a diagnostic counter; never index every target column for every source column.
4. `_drop_reverse_fk_refs_for_source(source_key)` removes old refs before a table's columns are replaced using only the source-forward map.
5. `_add_reverse_fk_refs_for_column(schema, table_name, col)` adds entries:
   ```lua
   {
     src_schema = schema,
     src_table = table_name,
     src_col = source_column,
     constraint = ref.constraint_name,
     ordinal = ref.source_ordinal,
     target_schema = ref.target_schema,
     target_table = ref.target_table,
     target_col = target_column,
     source_columns = ref.source_columns,
     target_columns = ref.target_columns,
   }
   ```
6. `_add_reverse_fk_refs_for_column` updates both:
   - `reverse_fk_refs_by_target_key[target_key]` for O(1) target lookup;
   - `reverse_fk_refs_by_source_key[source_key]` with target keys contributed by that source for O(refs-for-source) eviction.
7. Enforce caps while adding:
   - per-target cap 50 refs; store/count overflow for that target so docs can render `+N more`;
   - per-source cap 100 refs; skip additional refs from a degenerate source table and record truncation;
   - total cap 50k refs as last-line defense.
8. `_refresh_reverse_fk_refs_for_table(schema, table_name, cols)` drops then re-adds refs for that source table.

Integrate `_refresh_reverse_fk_refs_for_table` inside `_update_column_index` or immediately around it from `_store_columns`, preserving one source of truth.

Overflow recovery:

- If a later refresh/rebuild drops below the relevant cap, clear the affected overflow/truncation state.
- Emit at most one `vim.notify("dbee-lsp: reverse-FK index overflow; some referenced-by docs truncated", vim.log.levels.WARN)` per overflow episode; reset the warning flag only after the index returns below cap.
</action>

<action>
Add public cache API:

```lua
SchemaCache:get_reverse_fk_refs(schema, table_name, column_name, opts)
```

Required behavior:

1. Wrap the body in `epoch_authority.read_with_freshness(self, self.handler, self.conn_id, function(check) ... end)`.
2. Inside the freshness closure, first call `self:_fresh_lsp_scope()`. If it returns nil, `authority_unavailable`, or `stale_root_epoch`, return `{}`.
3. Caller passes already-resolved `schema/table/column` from `get_column_metadata`; this API does not perform a second lookup/canonicalization pass. It only canonicalizes those exact values into the fold-aware reverse-FK key.
4. Check `reverse_fk_index_epoch == check.cache_epoch`; return `{}` when stale/missing.
5. Filter returned refs to source schemas/tables admitted by the current `_fresh_lsp_scope()` authority result so filtered-out source tables do not leak through reverse docs.
6. Return an owned sorted array by `(src_schema, src_table, src_col, constraint, ordinal)`. `resolve.metadata_for` owns that array and must not deep-copy it again.
7. If a target/source/global cap truncated results, include truncation metadata on the returned array (for example `_truncated_count` and `_overflow = true`) so documentation can render a visible `+N more` line. Do not collapse all lookups to silent `{}` on overflow.
8. Add `reverse_fk_index_size`, per-target/source truncation counters, and `reverse_fk_index_overflow` to `get_stats()`.
</action>
