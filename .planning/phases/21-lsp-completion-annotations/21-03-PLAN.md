# 21-03 - Resolve Documentation Enrichment

<objective>
Append reverse-FK "Referenced by" documentation to resolved column completion items when the completed column is an FK target.
</objective>

<must_haves>
- Use the existing `completionItem/resolve` path.
- Preserve stale-generation and authority fail-closed behavior.
- Preserve richer column completion `detail`; do not downgrade it to bare type.
- Do not run DB metadata calls from resolve.
- Render at most 50 reverse refs per target column plus a visible truncation line.
- Avoid double deep-copy of reverse refs on the resolve hot path.
- Resolve memo keys use `reverse_fk_index_generation`, not `reverse_fk_cache_epoch`.
- Reverse-FK refs are sorted on read; the 50-ref cap bounds sort cost.
</must_haves>

<action>
Add `reverse_fk_index_generation` to the resolve memo key.

Reasoning:

- `resolve.lua` already prunes memo entries when `cache:generation()` changes.
- `_store_columns` bumps metadata generation whenever column metadata changes.
- However, column eviction can drop reverse-FK refs through `_drop_column_index` without bumping `cache:generation()` or changing `cache_epoch`.
- Therefore the resolve memo key must include `reverse_fk_index_generation`, the monotonic mutation counter for reverse-FK indexes, so cached docs cannot outlive a reverse-index shrink.

Required behavior:

1. Expose a cache method or stats field that lets resolve read the current `reverse_fk_index_generation` without bypassing freshness semantics.
2. Compose the memo key from the existing resolve dimensions plus `reverse_fk_index_generation`.
3. Keep the existing stale-generation/root-epoch rejection in `resolve.handle`; the new dimension is additive.
4. Behavioral test requirement: `invalidate()` followed by repopulation must produce fresh reverse docs and must not reuse an empty memo entry from generation 0.
5. Memo cardinality remains bounded because `resolve.lua` evicts stale-generation entries at access time; the new dimension is sub-cardinality of `cache:generation()` in steady state and changes only on reverse-index mutation.
6. Emit diagnostic marker `LSP21_DIAG_RESOLVE_MEMO_SIZE` after the eviction-churn fixture to expose memo cardinality without adding a strict marker.
</action>
