# 21-03 - Resolve Documentation Enrichment

<objective>
Append reverse-FK "Referenced by" documentation to resolved column completion items when the completed column is an FK target.
</objective>

<must_haves>
- Use the existing `completionItem/resolve` path.
- Preserve stale-generation and authority fail-closed behavior.
- Preserve richer column completion `detail`; do not downgrade it to bare type.
- Do not run DB metadata calls from resolve.
</must_haves>

<tasks>
<task id="21-03-T1" type="execute">
<read_first>

- `lua/dbee/lsp/resolve.lua:176-200`
- `lua/dbee/lsp/schema_cache.lua:2513-2552`
</read_first>
<action>
Enhance `metadata_for(cache, data)` in `resolve.lua` for `data.kind == "column"`:

1. Keep the existing `cache:get_column_metadata(...)` call.
2. If column metadata exists, call:
   ```lua
   cache:get_reverse_fk_refs(meta.schema, meta.table, meta.column, {
     schema_quoted = true,
     table_quoted = true,
     column_quoted = true,
   })
   ```
3. Attach a deep-copied list as `meta.referenced_by`.
4. If reverse refs are empty or unavailable, leave `meta.referenced_by` nil/empty.
5. Do not call reverse-FK API before stale-generation checks; `resolve.handle` already performs those checks before `metadata_for`.
</action>

<action>
Update `object_docs.lua` column formatting:

1. Add a local `format_referenced_by(refs, markdown)` helper.
2. Append this block after existing column metadata fields when `metadata.referenced_by` is non-empty:
   ```text
   Referenced by:
     - schema.table.column  (constraint: fk_name)
   ```
3. For composite refs, render the source column that pairs to the hovered target column. Do not duplicate the entire composite tuple for each target unless the source metadata lacks ordinals.
4. Use existing escaping helpers in markdown mode; simple identifiers still render exactly as the required text in tests.
5. Preserve plaintext fallback when completion clients request only plaintext documentation.
6. Update `format_resolve` so resolved column items keep the incoming richer `item.detail` when present. Do not replace `[PK]`/`[FK→...]` detail with bare type.
</action>
