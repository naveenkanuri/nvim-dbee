<objective>
Add live drawer search/filter (DRAW-01) to nvim-dbee.

Purpose: Users with large schemas (hundreds of tables) can quickly find objects by typing a filter pattern. The tree filters in real-time, showing only matching table/view/procedure/function nodes and their ancestor containers (connections, schemas).
Output: Filter action with NuiInput popup via menu.filter(), a separate restore snapshot and searchable cached model built from a shared `drawer/model.lua` layer, explicit mid-filter refresh policy, explicit buffer/layout teardown cleanup, deterministic accept/cancel behavior, headless tests, and a hard performance gate.

CRITICAL CONSTRAINTS:
1. Filter MUST operate on a searchable cached model assembled ONCE when filter starts from the ready cached corpus only — NEVER call `convert.handler_nodes()`, `convert.editor_nodes()`, `drawer_model.build_rendered_model()`, or `refresh()` from `apply_filter()` / live `on_change`
2. Filter popup MUST reuse drawer/menu.lua infrastructure (extend with menu.filter()) — do NOT create a parallel NuiInput popup path in init.lua
3. Filter MUST save/restore pre-filter cursor position on Esc, and explicitly expand/materialize the ancestor chain of the selected node on CR before placing cursor
4. Filter MUST search all ready cached structure data, including collapsed cached branches; search results must NOT depend on prior expansion history
5. Pressing `/` MUST NOT trigger `connection_get_structure_async()` or otherwise warm cold schema caches; cold-cache and cached-error connections are excluded from the searchable corpus instead
6. If zero ready cached connections exist, filter mode MUST WARN `No cached connections available for filter` and refuse to start. If only part of the drawer corpus is ready, the prompt MUST still open and visibly report `N of M connections cached`
7. While filter mode is active, refresh-driving events (`current_connection_changed`, `current_note_changed`, `structure_loaded`, `database_selected`) MUST cancel filter mode before refresh; stale snapshots may not be restored after new data arrives
8. The hard zero-RPC guarantee applies to typing only (`on_change` / `apply_filter`). Prompt INTERACTION mode may trigger the same RPC-backed expand/action behavior as the normal drawer
9. `bufhidden = "hide"` plus `DrawerUI:rebuild_buffer()` is the required drawer lifecycle contract. External buffer invalidation must still interrupt filter state safely before `show()` recreates the buffer
10. Debounce is NOT a substitute for a slow hot path. Raw apply cost must meet the performance gate before the plan can be considered complete.
11. Layout-owned drawer hide/close and full dbee shutdown MUST interrupt filter mode before destroying the host window, clearing prompt/timer/snapshot state instead of relying on implicit popup teardown.
</objective>

**1. drawer/menu.lua -- Add menu.filter() function (after the existing menu.input function, before `return M`):**

This extends the existing NuiInput popup infrastructure with `on_change` support for live filtering.

Interaction model for Phase 4 is explicit and conflict-free:
- Prompt starts in INSERT mode for typing.
- While in INSERT mode, only non-printing keys are forwarded to the drawer tree: `<Up>`, `<Down>`, and `<Tab>`/`<S-Tab>` for selection/toggle.
- `<CR>` ALWAYS means "accept filter" via the prompt's native submit behavior; it is never forwarded as `action_1`.
- `<C-]>` switches the prompt into INTERACTION mode (prompt normal mode) without unmounting.
- In INTERACTION mode, `j/k` and `<Up>/<Down>` move tree selection, the existing drawer `toggle`/`action_2`/`action_3` mappings are forwarded if they exist, `action_1` is available on an explicit non-conflicting alias `<C-y>`, and normal-mode `<CR>` is explicitly remapped back to submit the current prompt value (real prompt-buffer normal `<CR>` otherwise re-enters insert mode).
- Press `i` to return to INSERT mode and continue typing.
- Reserved prompt keys win over forwarded custom mappings while filter mode is active; conflicting user bindings are skipped instead of overriding submit/mode-switch/navigation behavior.
- Do NOT forward printable insert-mode keys like `o`, `cw`, or `dd` while the user is typing.

```lua
-- Live filter with on_change callback.
---@param opts { relative_winid: integer, coverage_label?: string, on_change: fun(value: string), on_submit: fun(value: string), on_close: fun(), mappings: key_mapping[], forward_insert: table<string, fun()>, forward_normal: table<string, fun()> }
function M.filter(opts)
  if not opts.relative_winid or not vim.api.nvim_win_is_valid(opts.relative_winid) then
    error("no window id provided")
  end

  local width = vim.api.nvim_win_get_width(opts.relative_winid)
  local input
  local manual_submit_value = nil

  local popup_options = {
    relative = {
      type = "win",
      winid = opts.relative_winid,
    },
    position = {
      row = 0,
      col = 0,
    },
    size = {
      width = width,
    },
    zindex = 160,
    border = {
      style = { "─", "─", "─", "", "─", "─", "─", "" },
      text = {
        top = " Filter ",
        top_align = "left",
        bottom = opts.coverage_label,
        bottom_align = "right",
      },
    },
    win_options = {
      cursorline = false,
    },
  }

  local function current_value()
    local line = vim.api.nvim_buf_get_lines(input.bufnr, 0, 1, false)[1] or ""
    if line:sub(1, 1) == "/" then
      return line:sub(2)
    end
    return line
  end

  input = NuiInput(popup_options, {
    prompt = "/",
    default_value = "",
    on_change = opts.on_change,
    on_submit = function(value)
      manual_submit_value = nil
      if opts.on_submit then
        opts.on_submit(value)
      end
    end,
    on_close = function()
      if manual_submit_value ~= nil then
        local value = manual_submit_value
        manual_submit_value = nil
        if opts.on_submit then
          opts.on_submit(value)
        end
        return
      end
      if opts.on_close then
        opts.on_close()
      end
    end,
  })

  local RESERVED_MAP_OPTS = { noremap = true, nowait = true }
  local FORWARDED_MAP_OPTS = { noremap = true, nowait = true }

  -- Map <Esc> in both insert and normal mode to unmount (NuiInput does not auto-unmount on Esc)
  input:map("i", "<Esc>", function()
    input:unmount()
  end, RESERVED_MAP_OPTS)
  input:map("n", "<Esc>", function()
    input:unmount()
  end, RESERVED_MAP_OPTS)

  -- INSERT mode: typing first, only non-printing navigation/toggle keys forwarded.
  for key, fn in pairs(opts.forward_insert or {}) do
    input:map("i", key, fn, FORWARDED_MAP_OPTS)
  end

  -- Enter prompt interaction mode without dismissing the filter.
  input:map("i", "<C-]>", function()
    vim.cmd("stopinsert")
  end, RESERVED_MAP_OPTS)

  -- INTERACTION mode: drawer actions/navigation are forwarded here.
  input:map("n", "i", function()
    vim.cmd("startinsert")
  end, RESERVED_MAP_OPTS)
  input:map("n", "<CR>", function()
    manual_submit_value = current_value()
    input:unmount()
  end, RESERVED_MAP_OPTS)
  for key, fn in pairs(opts.forward_normal or {}) do
    input:map("n", key, fn, FORWARDED_MAP_OPTS)
  end

  input:mount()
  return input
end
```

    return
  end

        return mapping.key
      end
    end

    if not is_single_prompt_safe_key(key) then
      return
    end

    if RESERVED_FILTER_KEYS[normalized] then
      return
    end
    map[normalized] = fn
  end

    self.pre_filter_expansion = nil
    self.pre_filter_cursor = nil
    return
  end

  local coverage_label = string.format("%d of %d connections cached", self.filter_cached_connections, self.filter_total_connections)

    coverage_label = coverage_label,
    forward_insert = {
      ["<Up>"] = function() move_tree_selection(-1) end,
      ["<Down>"] = function() move_tree_selection(1) end,
      ["<Tab>"] = function()

          expand_node_filter_safe(self, node)
        end
      end,

          node:collapse()
          self.tree:render()
        end
      end,
    },

        ["j"] = function() move_tree_selection(1) end,
        ["k"] = function() move_tree_selection(-1) end,
        ["<Up>"] = function() move_tree_selection(-1) end,

            perform_action(node.action_1)
          end
        end,

      maybe_add_forwarded(map, forwardable_mapping_key_for("expand"), function() ... end)
      maybe_add_forwarded(map, forwardable_mapping_key_for("collapse"), function() ... end)
      maybe_add_forwarded(map, forwardable_mapping_key_for("action_2"), function() ... end)
      maybe_add_forwarded(map, forwardable_mapping_key_for("action_3"), function() ... end)
      return map

        return

    end,
    on_submit = function(_)

        return
      end

      self:cancel_pending_filter_apply()
      local selected_node = self.tree:get_node()
      local selected_id = selected_node and selected_node:get_id()
      local selected_path = selected_path_ids(self.tree, selected_node)
      self.filter_text = ""
      self.filter_input = nil
      self:render_restore_snapshot()

      if selected_id then
        local ok_path, resolved_id = materialize_selected_path(self, self.tree, selected_path)
        self.tree:render()
        -- Focus exact selected_id when possible; otherwise warn and fall back to resolved_id.
      end

      self.filter_restore_snapshot = nil
      self.filter_search_model = nil
      self.pre_filter_expansion = nil
      self.pre_filter_cursor = nil
      self.filter_cached_connections = 0
      self.filter_total_connections = 0

        return

      self.filter_input = nil
      self:render_restore_snapshot()

      self.filter_search_model = nil
      self.pre_filter_expansion = nil
      self.pre_filter_cursor = nil

      self.filter_total_connections = 0
