- `R` keeps D-63 exactly, including `database_switch` rows resolving to the nearest ancestor connection.
- Use `focus_pane()` / `ensure_drawer_visible()` for layout interaction per D-70; do not assume a 4-pane layout.
- Keep filter close behavior exactly per D-31: restoring the pre-filter snapshot, not rebuilding from latest cache.

<interfaces>
Connection test contract:
```lua
vim.fn.DbeeConnectionTest(conn_id) -> nil|{ error_kind = "auth"|"network"|"driver"|"unknown", message = "..." }
handler:connection_test(conn_id) -> nil|{ error_kind = "...", message = "..." }
-- opens the adapter, may run a trivial liveness probe, closes it, and never changes
-- current_connection, source state, _struct_cache, or persisted files
```

Drawer dispatcher contract:
```lua
perform_action({
  mode = "close_only", -- authoritative invalidation will rerender later
  action = function(close_popup, select, input) ... end,
})

perform_action({
  mode = "refresh_after_action", -- pure UI action owns targeted refresh
  action = function(refresh_now, select, input) ... end,
})
```

Connection-only row metadata:
```lua
{
  id = conn.id,
  type = "connection",
  source_meta = {
    id = source_id,
    name = source_name,
    can_create = true,
    can_update = true,
    can_delete = true,
    file = "/path/to/source.json"|nil,
  },
}
```
